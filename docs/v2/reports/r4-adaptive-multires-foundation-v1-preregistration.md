# R4 Adaptive Multi-Resolution Foundation Preregistration

Date: 2026-06-17

Experiment ID: `r4-adaptive-multires-foundation-v1`

Contract: ADR 0154

Status: preregistered; implementation not yet authorized as evidence

## Question

Can a 61- or 91-cell focal-board near field, combined with explicitly ablated
far habitat, wildlife, and frontier topology, preserve every registered
long-range public distinction while using materially fewer model-visible
tokens than the exact R2 state?

## Immutable Inputs

- accepted 60,000-position R0/R2 corpus;
- `compact-entity-v2` source records;
- accepted R2 `SparsePublicState` semantics and codec;
- authoritative `cascadia-game` D6 transforms;
- AAAAA scoring-card metadata already present in the records; and
- no terminal target, teacher label, hidden order, future refill, or gameplay
  result as model input.

## Arms

Both `radius4-61` and `radius5-91` run:

1. `n0-near-only`;
2. `h-habitat`;
3. `w-wildlife`;
4. `f-frontier`;
5. `hw-habitat-wildlife`;
6. `hf-habitat-frontier`;
7. `wf-wildlife-frontier`;
8. `hwf-all-topology`; and
9. `e-exact-far-control`.

This is a complete three-factor H/W/F lattice plus one exact information
ceiling. No arm may silently add a feature block that its ID does not name.

## Exactness Before Measurement

Before corpus execution:

- every local index and inverse must pass for both disks;
- center selection must match the accepted R0 implementation;
- exact local plus overflow occupancy must reconstruct R2;
- packed decode and re-encode must be byte-identical;
- every malformed count, coordinate, ordering, varint, flag, radius, and
  trailing byte must fail closed;
- all twelve D6 transforms must round-trip exact semantics; and
- target mutation must leave exact and feature bytes unchanged.

Failure blocks production. Repair requires a new immutable source identity and
a fresh smoke.

## Distributed Execution

Each host owns one nonoverlapping train/validation part pair. Every host runs
both radii and all arms over its unique rows.

The production graph contains:

1. immutable source and release-binary fanout;
2. four host preflights;
3. four concurrent corpus shards;
4. checksum-bound collection;
5. forward aggregate;
6. reverse aggregate; and
7. byte-order proof.

Host loss may trigger work-conserving reassignment of the unstarted dataset
part. Completed rows are never duplicated in the scientific aggregate.

## Classification

`r4_adaptive_multires_foundation_passed`

- all mechanical, corpus, adversarial, compactness, and order gates pass.

`r4_adaptive_multires_information_insufficient`

- execution is valid, but HWF loses a required long-range distinction.

`r4_adaptive_multires_compactness_failed`

- exactness and information pass, but both compact HWF arms miss their token
  or packed-byte gates.

`r4_adaptive_multires_invalid`

- source, corpus, schema, exactness, D6, target-independence, shard-coverage,
  or order-proof evidence is malformed or incomplete.

Only the first classification authorizes MLX work.

## Claim Boundary

This experiment can establish:

- exact mechanical reconstruction;
- exact declared topology;
- information retention on frozen adversarial pairs;
- corpus token and byte distributions; and
- deterministic distributed execution.

It cannot establish:

- lower MLX latency;
- better ranking quality;
- search improvement;
- gameplay improvement; or
- achievement of the 100-point mean target.
