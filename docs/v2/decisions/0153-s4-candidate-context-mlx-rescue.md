# ADR 0153: S4 Candidate-Context MLX Rescue

Status: completed; rejected; all treatments degraded

Date: 2026-06-17

Experiment: `s4-candidate-context-mlx-comparison-v1`

Protocol: `s4-candidate-context-matched-comparison-v1`

Research-plan item: S4

Foundations:

- ADR 0150, R3 action-edit MLX comparison;
- ADR 0151, exact candidate-relation foundation; and
- ADR 0152, immutable candidate-context cache.

## Context

R3 closed the first learned compact-representation comparison with
classification `r3_action_edit_mlx_all_treatments_degraded`.

The full exact sparse R2 afterstate control achieved:

- R4800 MAE `1.3202285`;
- R4800 RMSE `1.7423126`;
- top-64 winner recall `0.725`;
- mean retained regret `0.09811535`;
- confidence coverage `0.9791667`;
- fixed-chunk throughput `86,208.07` actions per second;
- complete-decision P99 `130.947` milliseconds;
- peak active MLX memory `186,160,884` bytes; and
- peak process RSS `795,394,048` bytes.

The smallest R3 arm, radius one plus exact global edits, improved aggregate
top-64 recall to `0.7458333` but regressed value error, protected slices,
confidence coverage, throughput, latency, and memory. Its R4800 MAE was
`1.4885582`, RMSE was `1.9458030`, fixed-chunk throughput was `56,036.67`
actions per second, and peak process RSS was `1,459,355,648` bytes.

Radius one is therefore not an accepted representation. It is retained here
only as a deliberately failed substrate for a narrow rescue question:

> Did the compact arm fail because independently scored candidates lacked
> whole-decision candidate context, and can exact context recover the full-R2
> quality envelope without destroying serving viability?

This distinction is binding. S4 cannot promote radius one by comparing only
against another radius-one model.

## State-Footprint Correction

The earlier 441-cell dense state is oversized. The state-footprint census also
showed that 121 is not the cardinality of a complete centered hex disk:

```text
radius 5 = 91 cells
radius 6 = 127 cells
```

Exact compact regular representations therefore use a 91- or 127-cell window
plus explicit overflow, or a sparse occupied/frontier/action representation.
The R3 control was already sparse and averaged only 58.79 candidate tokens
with P99 85 and maximum 88. S4 is not a dense 441-versus-121 benchmark; it is
a test of candidate-set context over the failed sparse radius-one edit stream.

## Decision

Run one four-arm, iso-graph, iso-data, iso-initialization MLX comparison:

| Arm | Candidate-context treatment | Host |
|---|---|---|
| `c0-independent` | No candidate-set residual | `john1` |
| `t1-inducing-16` | Sixteen inducing latents over 256 anchors | `john2` |
| `t2-exact-relations` | Bounded exact relation-segment aggregation | `john3` |
| `t3-combined` | Inducing context plus exact relation segments | `john4` |

All arms use the same radius-one R3 candidate representation, exact global
edits, parent state, factual features, exact semantic supply, candidate
anchors, optimization schedule, validation corpus, and serving benchmark.

Every arm contains the same module graph and parameter tensors. Two frozen
scalar treatment gates choose whether the inducing and relation residuals are
active. Arm names do not change parameter count, layout, or initialization.

Every arm starts from the exact final checkpoint of the failed R3 radius-one
arm:

```text
  05befa5fae0c5f4af4f43f4b017ef74c1810b33f39b84bd8251c5f2a1a6e1919
```

New context output projections are zero initialized. Before optimization,
every S4 arm must produce byte-identical scores and standard errors to the
frozen R3 radius-one model on the same validation batch.

The base model is jointly fine-tuned in all four arms. Freezing the base would
answer a different question and is not permitted.

## Exact Shared Context

Every arm consumes candidate-context cache:

`fd3dcc8018cfe4b735a9a6514555e90e938fd142e746dc6d791f482e96463def`

It covers:

| Split | Groups | Candidates |
|---|---:|---:|
| Train | 560 | 280,012 |
| Validation | 240 | 860,203 |

Each decision exposes at most 256 observable anchors and six exact public
relations:

1. shared staged draft;
2. shared frontier destination;
3. shared wildlife destination;
4. equivalent public afterstate;
5. public dominance; and
6. sibling plan.

Each query-relation pair retains at most eight anchor siblings. Missing
neighbors are explicit. Relation graphs may not be regenerated by individual
arms or epochs.

## Shared Architecture

The accepted R3 parent and candidate encoders are reused. S4 adds:

- one 64-wide anchor representation per retained anchor;
- sixteen learned 64-wide inducing latents;
- one inducing-to-anchor and one query-to-inducing attention stage;
- six relation-specific bounded neighbor aggregators;
- exact relation sibling-count features;
- one gated inducing residual;
- one gated relation residual; and
- zero-initialized residual output projections.

The 256 anchors are encoded once per decision. Every complete action remains a
query and is scored in chunks of 256. Context never prunes or substitutes for
the complete candidate set.

## Validation

Each arm scores all 240 open validation decisions and all 860,203 actions once.
Required evidence includes:

- R4800 MAE, RMSE, bias, correlation, slope, and intercept;
- stable top-1/8/32/64 winner recall;
- top-1/8/32/64 retained regret;
- top-64 95% confidence-set coverage;
- early, middle, late, low-supply, and independent-winner slices;
- fixed action-hash prediction panel;
- one parent encode and one anchor encode per decision;
- complete optimization trace and checkpoint identity; and
- fresh-process serving measurements.

## Serving Benchmark

Each final checkpoint is reloaded in a fresh process. The benchmark excludes
five warmup iterations and measures 30 steady iterations.

It reports:

- fixed 256-action throughput and latency;
- 20 complete-decision throughput and P50/P95/P99 latency;
- one parent and one anchor encode per complete decision;
- MLX active and peak memory;
- process peak RSS and process swap;
- system swap before and after;
- request and result content addresses;
- checkpoint model BLAKE3;
- open-data verification ID; and
- context-cache ID.

Absolute gates:

```text
all validation actions scored exactly once
all outputs finite
one parent and one anchor encode per decision
process swap = 0
peak active memory <= 4 GiB
peak process RSS <= 4 GiB
complete-decision P99 <= 250 ms
fixed-chunk throughput >= 20,000 actions/s
```

The relative serving rescue gates against the R3 full-R2 control are:

```text
throughput >= 0.25x full-R2
P99 latency <= 1.75x full-R2
active memory <= 2.50x full-R2
RSS <= 1.50x full-R2
```

These looser relative gates acknowledge the explicit 256-anchor context cost.
They do not replace the absolute serving envelope.

## Classification

Classification precedence:

1. `s4_candidate_context_mlx_invalid_evidence`
2. `s4_candidate_context_mlx_control_failed`
3. `s4_candidate_context_mlx_all_treatments_degraded`
4. `s4_candidate_context_mlx_context_null`
5. `s4_candidate_context_mlx_context_signal_only`
6. `s4_candidate_context_mlx_compact_rescue_selected`

A treatment is noninferior to the S4 independent control only if:

```text
MAE delta <= 0.05
RMSE delta <= 0.05
top-64 recall delta >= -0.005
top-64 retained-regret delta <= 0.005
low-supply top-64 recall delta >= -0.01
independent-winner top-64 recall delta >= -0.01
confidence coverage >= 0.99
```

A treatment has a material context effect if at least one is true:

```text
MAE reduction >= 0.05
RMSE reduction >= 0.05
top-64 recall gain >= 0.02
top-64 retained-regret reduction >= 0.02
confidence coverage gain >= 0.01
```

`compact_rescue_selected` additionally requires every quality-noninferiority
gate against the R3 full-R2 control and every relative serving rescue gate.

Under the frozen full-R2 metrics, the quality rescue envelope is approximately:

```text
MAE <= 1.37023
RMSE <= 1.79231
top-64 recall >= 0.72000
retained regret <= 0.10312
low-supply recall >= 0.90228
independent-winner recall >= 0.79952
confidence coverage >= 0.99
```

The selected rescued arm is ordered by:

1. highest top-64 recall;
2. lowest top-64 retained regret;
3. lowest R4800 MAE;
4. highest fixed-chunk throughput; and
5. frozen arm order.

Forward and reverse report orders must produce byte-identical classification
files and a separate order proof.

## Consequences

The production comparison completed on all four hosts with classification
`s4_candidate_context_mlx_all_treatments_degraded`.

Exact relations produced the only material signal: top-64 recall improved by
2.08 points, low-supply recall by 5.26 points, and middle-phase recall by 4.94
points versus the matched independent control. It still failed
matched-control noninferiority because independent-draft recall regressed by
4.76 points and confidence coverage reached only 97.50%.

No treatment recovered the full-R2 quality envelope. All treatments also
failed the full-R2-relative complete-decision latency, active-memory, and RSS
gates. The independent S4 control itself degraded from the failed R3
radius-one warm start after the additional 3,000-step joint fine-tune.

Consequently:

1. No S4 arm advances to gameplay or promotion.
2. Radius one remains a failed representation.
3. The present 256-anchor inducing and relation-segment architecture is closed
   as a compact-rescue path.
4. Exact relation facts remain useful feature evidence for narrower R4, S3,
   and S5 ablations.
5. Exact sparse full R2 afterstates remain the quality and serving reference.
6. The primary representation lane moves to R4 adaptive multi-resolution
   foundations with exact overflow.
7. Future 91- or 127-cell dense-window experiments remain separate hypotheses
   and must retain exact overflow.
